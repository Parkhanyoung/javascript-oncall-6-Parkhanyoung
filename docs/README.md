### 핵심 정의

- 평일 및 휴일 비상근무 플랜을 짠다.

### 구현 사항

- [x] 멤버 구현
  - [x] 이름은 최대 5자이하이다.
- [ ] 멤버 콜렉션 구현
  - [ ] 최소 5명, 최대 35명이다.
  - [ ] 닉네임 중복은 불가하다
- [ ] 플래너 구현
  - [ ] 월을 입력받아 해당 월에 날짜에 맞게 근무자를 배정(나열)한다.
  - [ ] 연속 이틀 근무는 불가하다.

### 추가 구현 및 수정 사항

- [ ] 잘못된 입력 값 재입력 받기
- [ ] 컨트롤러 및 15줄 이상 로직 분리
- [ ] 상수 분리
- [ ] static, private 메소드 확인
- [ ] 네이밍 통일
- [ ] 중복 로직 분리

### 정의 모델

- 멤버: 이름
- 멤버콜렉션: 멤버 리스트
- 플래너

### 구현 과정

- [ ] 돌아가는 프로그램을 완성한다.
- [ ] 테스트 코드를 작성한다.
- [ ] 유효성 검증(예외 처리)을 추가 및 보완한다.
- [ ] 리팩토링 실시 (클린 코드 규칙 기반으로 검토)
  - [ ] 메소드 하나에 15줄이 넘어가지 않는지
  - [ ] depth가 2 이하인지
